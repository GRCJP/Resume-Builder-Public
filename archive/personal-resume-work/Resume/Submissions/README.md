# Submissions Folder

## Purpose
This folder contains **job-specific customized resumes** tailored for individual applications. These are working copies you modify for each job.

## Naming Convention

Use this format for easy tracking:

```
[Date]_[Company]_[Role]_[Version].docx
```

### **Examples:**
- `2025-11-22_TechCorp_GRC-Engineer_v1.docx`
- `2025-11-22_FinServ_Senior-GRC-Analyst_v1.docx`
- `2025-11-23_CloudCo_GRC-Engineer_v2.docx`

### **Why This Format:**
- Date first = Easy to sort chronologically
- Company name = Know where you applied
- Role = Remember which position
- Version = Track iterations

## Workflow

### **For Each Application:**

1. **Copy Master Resume**
   - Go to `Updated Resumes/` folder
   - Copy the best master version for this job type
   - Paste into `Submissions/` folder
   - Rename with date, company, role

2. **Analyze Job**
   - Open GRC Resume Builder tool
   - Upload the copied resume
   - Paste job description
   - Note match score and missing keywords

3. **Customize**
   - Open the submission resume
   - Add missing keywords naturally
   - Reorder bullets to emphasize what they want
   - Adjust summary to mirror job language
   - Save changes

4. **Re-Analyze**
   - Upload updated version to tool
   - Re-analyze against job description
   - Verify 80%+ match score
   - Make final tweaks if needed

5. **Submit**
   - Save final version
   - Export as PDF if needed
   - Submit application
   - Note in tracking spreadsheet

## Version Control

### **If You Need to Revise:**
- Save as v2, v3, etc.
- Example: `2025-11-22_TechCorp_GRC-Engineer_v2.docx`
- Keep v1 as backup
- Note in tracking spreadsheet which version you submitted

### **When to Create v2:**
- Recruiter asks for changes
- You realize you missed something
- Match score was too low
- Want to try different approach

### **Weekly:**
