## Folder Purposes

### **ğŸ“ Updated Resumes** (Master Versions)
**Purpose:** Store your main resume versions that hold all your information

**What goes here:**
- Your master/primary resume
- Specialized versions (Federal, Commercial, Engineering)
- Updated versions as you add achievements
- Source of truth for all applications

**When to update:**
- New certification earned
- Major project completed
- New achievement with metrics
- Quarterly refresh

**DO NOT:**
- Customize for specific jobs (use Submissions instead)
- Keep too many versions (3-5 max)
- Let them get stale

---

### **ğŸ“ Submissions** (Job-Specific Versions)
**Purpose:** Store customized resumes for individual job applications

**What goes here:**
- Resumes tailored for specific jobs
- Company-specific variations
- Submitted applications

**Naming format:**
```
[Date]_[Company]_[Role]_[Version].docx
Example: 2025-11-22_TechCorp_GRC-Engineer_v1.docx
```

**When to create:**
- For each job application
- When customizing for specific role
- When testing different approaches

**DO NOT:**
- Store master versions here
- Use generic file names
- Let folder get cluttered

---

### **ğŸ“ Templates** (Reference Only)
**Purpose:** Store formatting templates and examples

**What goes here:**
- ATS-friendly templates
- Formatting examples
- Reference samples

**DO NOT:**
- Edit these (they're references)
- Store your actual resumes here
- Delete (you might need them)

---

### **ğŸ“ Archive** (Old Versions)
**Purpose:** Store old versions and backups

**What goes here:**
- Original resumes
- Old drafts
- Previous versions
- Backups

**When to archive:**
- After creating new master version
- When cleaning up
- Monthly maintenance

## Complete Workflow

### **Step 1: Maintain Master Resumes**
Location: `Updated Resumes/`

1. Keep 3-5 master versions updated
2. Add new achievements quarterly
3. Update metrics as they improve
4. Version control with dates

### **Step 2: Apply for Jobs**
Location: `Submissions/`

1. Copy appropriate master from `Updated Resumes/`
2. Rename: `YYYY-MM-DD_Company_Role_v1.docx`
3. Customize for specific job
4. Test with GRC Resume Builder
5. Submit when 80%+ match
6. Track in spreadsheet

### **Step 3: Track Results**
Location: Tracking spreadsheet

1. Record each submission
2. Note match score
3. Track responses
4. Learn what works

### **Step 4: Update Masters**
Location: `Updated Resumes/`

1. Review successful submissions
2. Identify winning approaches
3. Update master versions
4. Archive old versions

## Example: Complete Application Process

### **Scenario: Applying to TechCorp for GRC Engineer**

**1. Prepare (Updated Resumes folder)**
- Open `Jonathan_Perez_GRC_Engineering.docx`
- Verify it's current
- Copy file

**2. Customize (Submissions folder)**
- Paste into `Submissions/2025-11/`
- Rename: `2025-11-22_TechCorp_GRC-Engineer_v1.docx`
- Open file

**3. Analyze (GRC Resume Builder)**
- Upload resume to tool
- Paste TechCorp job description
- Get match score (e.g., 75%)
- Note missing keywords: "Kubernetes", "container security"

**4. Edit (Submissions folder)**
- Open `2025-11-22_TechCorp_GRC-Engineer_v1.docx`
- Add Kubernetes and container security naturally
- Reorder bullets to emphasize automation
- Save changes

**5. Re-Analyze (GRC Resume Builder)**
- Upload updated version
- Re-analyze
- New match score: 92%
- Ready to submit!

**6. Submit**
- Export as PDF if needed
- Submit application
- Add to tracking spreadsheet

**7. Track**
| Date | Company | Role | File | Score | Status |
|------|---------|------|------|-------|--------|
| 11/22 | TechCorp | GRC Engineer | 2025-11-22_TechCorp_GRC-Engineer_v1.docx | 92% | Applied |

**8. Learn**
- If you get interview: Note what worked
- If no response: Try different approach next time
- Update master if you found winning formula

## File Naming Standards

### **Master Resumes (Updated Resumes folder):**
```
Jonathan_Perez_GRC_[Type].docx

Examples:
- Jonathan_Perez_GRC_Master.docx
- Jonathan_Perez_GRC_Federal.docx
- Jonathan_Perez_GRC_Commercial.docx
- Jonathan_Perez_GRC_Engineering.docx
```

### **Submissions (Submissions folder):**
```
YYYY-MM-DD_[Company]_[Role]_v[#].docx

Examples:
- 2025-11-22_TechCorp_GRC-Engineer_v1.docx
- 2025-11-22_FinServ_Senior-GRC-Analyst_v1.docx
- 2025-11-23_CloudCo_GRC-Engineer_v2.docx
```

### **Archives (Archive folder):**
```
[Original-Name]_[Date].docx

Examples:
- Jonathan_Perez_GRC_Master_2025-10.docx
- Jonathan_Perez_GRC_Federal_2025-09.docx
```
